Headless Engine

The HLS player
without opinions.

A headless HLS video player engine that works with any JavaScript framework — or no framework at all. Handles playback, quality switching, live streams, authentication, and secure fullscreen.

Installation
# install with npm, yarn, pnpm, or bun
npm install @nurav/player-core
yarn add @nurav/player-core
pnpm add @nurav/player-core
bun add @nurav/player-coreshell

No peer dependencies required. Works in any JS project — React, Vue, Svelte, vanilla, etc.

Quick start
// HTML
<video id="my-video" controls></video>

const video = document.getElementById("my-video");
const player = new Player({
  video,

});ts

Playback controls

All the methods you need to control playback:

// Play / pause
await player.play();
player.pause();
await player.togglePlay();

// Seek
player.seek(150);           // Jump to 2m30s
player.seekToLive();      // Jump to live edge

// Volume
player.setVolume(0.5);   // 0.0 – 1.0
player.mute();
player.unmute();

// Playback speed
player.setPlaybackRate(2);  // 2× speed

// Quality
player.setQuality("auto");  // Let hls.js decide
player.setQuality(2);      // Pick level by IDts

Events API

The player is an EventEmitter. Subscribe to specific events: 

player.on("play",         (state) => { /* playing */ });
player.on("pause",        (state) => { /* paused */ });
player.on("timeupdate",   (state) => { /* time ticked */ });
player.on("ended",        (state) => { /* finished */ });
player.on("qualitychange",(quality) => { /* new quality */ });
player.on("error",        (error) =>  { /* something broke */ });
player.on("destroy",      () =>       { /* player torn down */ });ts

Token authentication

For streams requiring authentication (e.g. Akamai tokenized URLs), provide a tokenFetcher function: 

import type { TokenFetcher } from "@nurav/player-core";

const tokenFetcher: TokenFetcher = async ({ src, signal }) => {
  const res = await fetch("/api/get-token", {
    method: "POST",
    body: JSON.stringify({ url: src }),
    signal,
  });
  const data = await res.json();

  return {
    url:       data.signedUrl,   // Signed stream URL
    expiresIn: data.expiresIn,   // Optional: seconds until expiry
    headers:   data.headers,     // Optional: custom HTTP headers
  };
};

const player = new Player({ video, src, tokenFetcher });ts
↻

The player automatically refreshes the token before it expires when you provide expiresIn in the return value.

Live streams
const { isLive, liveLatency, isAtLiveEdge } = player.getState();

// Jump to the live edge
player.seekToLive();ts

Live edge detection uses actual latency with hysteresis — not timestamp snapshots:

Latency Badge Behavior
0 – 2.5s 🔴 LIVE Within half the sync threshold
2.5 – 5s ● (grey) Hysteresis zone — prevents badge ping-pong
5s+ Go Live Behind edge; click to catch up
ℹ

The badge updates via a 1-second polling interval even while paused, so pausing will correctly show "Go Live" once you fall behind.

Architecture

The player is composed of 8 specialized managers, each with a single responsibility: 

            ┌──────────────┐
            │  Your Code   │
            └──────┬───────┘
                   │ Player API
      ┌────────────▼───────────────┐
      │          Player             │
      └──┬───┬───┬───┬───┬─────────┘
         │   │   │   │   │
     HlsMgr  Store Auth Live  Fullscreen
     Network  Error  Keyboard  Security
HlsManager
hls.js init, error recovery, quality switching
LiveManager
Live edge detection, DVR mode, pause polling
ErrorManager
HTTP and stream error classification and recovery
AuthManager
Token fetch, refresh, header injection
NetworkManager
Online/offline detection + auto-retry
FullscreenManager
Fullscreen API across browsers
KeyboardManager
Keyboard shortcuts (Space, arrows, F, M…)
SecurityManager
DevTools traps, F12 shields, context menus, auto-resume
ℹ

Managers receive callbacks (e.g. onFatalError, onLevelUpdated) instead of accessing EventEmitter.emit() directly, making each manager independently testable.

Security manager

The SecurityManager provides enterprise-grade protection for intellectual property and secure broadcast streams. Enable it via security.disableDevOptions: 

const player = new Player({
  video,
  src: "https://example.com/stream.m3u8",
  security: { disableDevOptions: true },
});ts

When active, the player enables multiple defense layers:

Layer What it does
Event interception Blocks F12, view source, inspect, right-click, drag
CSS protection Injects user-select: none, pointer-events: none on video
Resize traps Detects docked DevTools panels wider than 250px
Debugger timing Runs debugger timing loops; locks if thread halted >200ms
Lock screen Pauses video, blurs with glassmorphic security overlay
Auto-resume Clears lock and resumes playback when DevTools close
↻

The player automatically resumes streaming when developer tools are closed — no manual action required from the user.

      <button onClick={() => playerRef.current?.seekToLive()}>
        Go Live
      </button>
    </>
  );
}tsx

Headless React Hook: useHlsPlayer

Skip the built-in UI entirely and build your own from scratch: 

import { useHlsPlayer } from "@nurav/player-react";

function CustomPlayer() {
  const { player, state, error, rootRef, videoRef } = useHlsPlayer({
    src: "https://example.com/stream.m3u8",
    autoPlay: true,
  });

  return (
    <div ref={rootRef} style={{ width: 640, height: 360, position: "relative" }}>
      <video ref={videoRef} style={{ width: "100%", height: "100%" }} />
      {state?.isPlaying ? "▶ Playing" : "⏸ Paused"}
      {error && <p>Error: {error.message}</p>}
    </div>
  );
}tsx
